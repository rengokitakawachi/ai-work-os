# 11_Doc_Style: ドキュメント書式ルール

## 1. 目的

本ドキュメントは、AI Work OS における docs の記述ルールを定義する。

目的は以下の3点

・AIが安定して構造を解析できること
・差分（Diff）が最小になること
・人間とAIが共同で編集しても崩れないこと

---

## 2. 基本原則

```
SSOT = docs
```

```
構造は固定、内容のみ更新
```

```
差分最小で更新する
```

---

## 3. 見出しルール

### 3.1 見出し階層

使用する見出しは以下のみ

# 大項目

## 中項目

### 小項目

#### は使用しない

---

### 3.2 見出し命名

・名詞で統一する
・動詞は禁止
・簡潔にする

例

正
Task定義
API仕様

誤
Taskを定義する
APIについて

---

## 4. セクション構造

各ドキュメントは以下の順序で構成する

1
目的

2
定義

3
構造

4
処理

5
例

6
補足

※不要なセクションは省略可

---

## 5. 箇条書きルール

### 5.1 記号

・ハイフン「-」を使用する
・全角記号は禁止

---

### 5.2 文体

・体言止めまたは短文
・1行1意味

例

正
・Taskは30〜120分で完了する単位
・Next Actionは動詞で始める

誤
・Taskは基本的に短時間で終わるように設計されるべきである

---

## 6. コードブロックルール

### 6.1 使用方法

コードは必ずコードブロックで記述する

```
コード
```

---

### 6.2 ネスト禁止

コードブロック内に ``` を書かない

---

### 6.3 JSON表記

JSONは整形する

{
"key": "value"
}

---

## 7. 命名規則

### 7.1 ファイル名

形式

```
NN_name.md
```

例

01_concept.md
03_api_spec.md

---

### 7.2 API命名

```
/api/{resource}/{action}
```

例

/api/task/create
/api/docs-read

---

## 8. 空行ルール

・セクション間は1行空ける
・箇条書きは詰める
・過剰な空行は禁止

---

## 9. 更新ルール（最重要）

AIおよび人間は以下を厳守する

```
既存文章を削除しない
```

```
構造を変更しない
```

```
差分最小で追記・修正する
```

---

## 10. 禁止事項

・構造の大幅変更
・見出し階層の変更
・文体の混在
・推測による追記

---

## 11. AI向けルール

AIは以下を必ず守ること

1
更新前に対象ドキュメント全文を読む

2
部分修正で済む場合は全体を書き換えない

3
未確定事項は docs に書かない

4
必要に応じて notes に記録する

---

## 12. 完成状態の定義

ドキュメントが正しく書かれている状態とは

・見出し構造が統一されている
・命名が一貫している
・差分が最小で更新されている
・AIが構造を誤解しない

である
