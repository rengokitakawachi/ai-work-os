# 07_External_Integration: 外部システム連携方針

1. Outlook 連携
- 役割: 予定の正本（SSOT）。
- 用途: 今日の状況把握、空き時間確認、Daily Plan 生成。
- v1 では参照（Read）を中心に実装し、AI がスケジュールを把握できるようにする。

2. Teams 連携
- 役割: リアルタイム通知と連絡。
- 用途: 日報サマリの通知、週報の共有、重要タスクのプッシュ通知。
- 実装時期: Phase 2 (v1.5) を予定。

3. kintone 連携
- 役割: 案件管理・進捗管理の正本。
- 用途: 組織としての案件ステータス管理。
- 原則: 案件（kintone） ↔ タスク（Todoist） ↔ 知識（Obsidian）の役割を分離する。
- 実装時期: 未解決課題（Backlog）とし、同期プロトコルを別途策定する。

4. セキュリティ・ガバナンス
- API キーはすべて Vercel の環境変数で管理する。
- AI UI（Gemini/ChatGPT）には生のリポジトリ権限を直接渡さず、Vercel API を介して最小権限で操作する。
- 知識の主利用はローカル Obsidian で行い、AI は正本を保持しない。
