# 01_Concept: 基本理念と仕事の循環

## 1. AI Work OS の定義
AI を中核 UI とし、個人の知識、戦略、実行、レビューをシームレスに統合するワークシステム。

## 2. 仕事の基本循環
システムは以下のサイクルを回すことを目的とする。
Knowledge（知識）
↓
Strategy（戦略）
↓
Execution（実行）
↓
Review（レビュー）
↓
Improve（改善）

## 3. 9つの基本原則
1. 原則1: AIは思考パートナーである。単なるツールではなく、分析や要約、タスク分解の壁打ち相手として活用する。
2. 原則2: 知識→戦略→実行の順で仕事を設計する。場当たり的な実行ではなく、知識を戦略に変えてから動く。
3. 原則3: タスクは Next Action で管理する。GTD の Capture/Clarify プロセスを AI が支援し、具体的な行動へ変換する。
4. 原則4: タスク粒度は 30分〜120分 とする。これを超えるものは分解し、小さすぎるものはまとめる。
5. 原則5: 入力時の摩擦を最小化する。インテリジェント・キャプチャにより、ユーザーは分類を気にせず Inbox へ投げ、AI が属性を推定する。
6. 原則6: 知識は Obsidian (Markdown) に保存する。特定のツールに依存せず、将来にわたって資産として活用可能にする。
7. 原則7: 情報の正本 (SSOT) を明確にする。今やることは Todoist、知見は Obsidian、ソースは GitHub と分離する。
8. 原則8: レビューによってシステムを進化させる。日報・週報・統合レビューを通じて、ボトルネックを特定し改善する。
9. 原則9: 段階的に進化させる。手動運用と自動化を組み合わせ、継続可能な範囲で機能を拡張する。

## 4. ユーザー制約（木下スタイル）
1. 文体制約: プレーンテキスト優先。装飾（太字や過度な記号）は一切使用しない。再処理や転用を容易にするため。
2. 思考の外部化: 計画の立案やタスク分解は AI との対話形式で行う。
3. レビューの重視: 実績記録は日報で行い、週報を統合レビューの起点とする。
