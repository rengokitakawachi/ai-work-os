# 12_ai_update_policy.md

## 目的

本ドキュメントは、AI が docs を更新する際のルールと制約を定義する。

AI による更新は、本ポリシーに従って厳格に制御される。

---

## 前提 / 定義

docs は AI Work OS の正式仕様書（SSOT）である。

AI は docs を最優先の情報源として扱う。

AI は docs の整合性を維持する責任を持つ。

AI にとっての docs 取得成功は、API レスポンスの data.content を実際に参照できた状態とする。

ok: true のみでは取得成功とみなさない。

docs-read は本文取得の正本 API とする。

---

## 基本原則

AI は docs を更新する前に、必ず docs 本文を取得し理解する。

AI は docs の整合性を維持する責任を持つ。

AI は差分最小で更新する。

AI は未確定事項を docs に書かない。

AI は会話や推測ではなく、取得した docs 本文を根拠に判断する。

---

## 処理（フロー）

### 更新前の必須行動

AI は docs を更新する前に、必ず以下を実行する。

1  
GET /api/docs により docs 一覧を取得する

2  
対象ファイルを /api/docs-read で取得する

3  
必要に応じて関連 docs も取得する

4  
現在の仕様を正確に理解する

5  
必要に応じて対象 docs を再取得する

---

### 取得判定

AI は以下の条件を満たした場合のみ、本文取得成功と判定する。

- レスポンスが ok: true
- data.content が存在する
- data.content が空ではない
- status が OK である

上記を満たさない場合は、更新処理に進んではならない。

---

### 再取得ルール

以下の場合、AI は再取得を行う。

- data.content を確認できない場合
- data.content が空の場合
- status が OK でない場合
- 情報が不明確な場合
- 更新直前に最新化が必要な場合

---

## ルール（制約・禁止事項）

### 更新ルール

AI は次のルールを必ず守る。

- 既存文章を維持する
- 差分最小で修正する
- 不要な構造変更を行わない
- 命名を変更しない
- 未確定事項を追加しない
- docs 未取得状態で更新しない
- ok: true のみで更新可と判断しない
- data.content 未確認で更新しない

---

### 禁止事項

AI は次の行為を行ってはならない。

- docs を読まずに更新すること
- 推測で仕様を書き換えること
- 複数ファイルにまたがる大規模変更を一度に行うこと
- 既存構造を破壊する変更を行うこと
- notes の内容を docs より優先すること
- docs 未取得での修正
- 本文未取得での修正
- 「アクセスできない」と判断すること

---

### 更新対象の制約

AI は以下の範囲でのみ更新を行う。

- 軽微な整合修正
- 誤記修正
- 構造を維持した追記
- 明確に必要な不足情報の補完

---

## 補足

### notes との関係

notes は検討メモであり、SSOT ではない。

AI は notes を参考情報として扱う。

notes の内容を docs に反映する場合は、仕様として確定していることを前提とする。

---

### 出力ルール

docs 更新時は、必ずファイル全文を出力する。

部分修正ではなく、完成形として提示する。

コードブロック外に本文を出してはならない。

---

### 最重要原則

SSOT = GitHub docs

docs は常に正確であるべき

AI は docs の整合性を最優先で維持する
