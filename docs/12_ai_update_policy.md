# 12_ai_update_policy.md

## 概要

本ドキュメントは、AI が docs を更新する際のルールと制約を定義する。

AI による更新は、本ポリシーに従って厳格に制御される。

---

## 基本原則

docs は AI Work OS の正式仕様書（SSOT）である。

AI は docs を最優先の情報源として扱う。

AI は docs の整合性を維持する責任を持つ。

---

## 更新前の必須行動

AI は docs を更新する前に、必ず以下を実行する。

対象ファイルを /api/docs-read で取得する

必要に応じて関連 docs も取得する

現在の仕様を正確に理解する

---

## 更新ルール

AI は次のルールを必ず守る。

既存文章を維持する

差分最小で修正する

不要な構造変更を行わない

命名を変更しない

未確定事項を追加しない

---

## 禁止事項

AI は次の行為を行ってはならない。

docs を読まずに更新すること

推測で仕様を書き換えること

複数ファイルにまたがる大規模変更を一度に行うこと

既存構造を破壊する変更を行うこと

notes の内容を docs より優先すること

---

## 更新対象の制約

AI は以下の範囲でのみ更新を行う。

軽微な整合修正

誤記修正

構造を維持した追記

明確に必要な不足情報の補完

---

## notes との関係

notes は検討メモであり、SSOT ではない。

AI は notes を参考情報として扱う。

notes の内容を docs に反映する場合は、仕様として確定していることを前提とする。

---

## 更新手順

AI は docs 更新時に以下の手順を守る。

1  
対象 docs を取得する

2  
差分内容を明確にする

3  
差分最小で修正文を生成する

4  
更新内容を一貫した形式で出力する

---

## 出力ルール

docs 更新時は、必ずファイル全文を出力する。

部分修正ではなく、完成形として提示する。

---

## 最重要原則

SSOT = GitHub docs

docs は常に正確であるべき

AI は docs の整合性を最優先で維持する
