# 05_Roadmap: 開発ロードマップと未解決課題

## 1. 開発フェーズ (Milestones)

### Phase 1: Prototype / MVP (現在のフェーズ)
- [x] 概念設計とアーキテクチャの確定
- [x] Vercel API 経由での Todoist タスク登録の実装
- [ ] GitHub API を使用した Markdown 出力（ナレッジ連携）の実装

### Phase 2: Knowledge Integration
- [ ] Obsidian 内データの AI 検索連携（RAG 基礎）
- [ ] Teams / Outlook との通知・ログ連携

### Phase 3: Analysis & Optimization
- [ ] 想定時間と実績時間の乖離を記録するログ機能
- [ ] AI による作業見積もりの自動補正アルゴリズムの実装

### Phase 4: Full AI Work OS
- [ ] 全デバイス（スマホ・PC・音声）からの収集環境の完全統合
- [ ] AI による中長期戦略支援の自動化

## 2. レビュー設計 (Review System)

### 2.1. 日報 (Daily Report)
- 今日のタスク実施状況、所要時間、発生した問題の確認と記録。

### 2.2. 週報 (Weekly Report)
- 今週の成果、課題、来週の重点項目の整理。

### 2.3. 統合レビュー
- 戦略上の問題点とシステム上のボトルネックをAIが分析し、改善案を提示。

## 3. 未解決課題 (Backlog)
- **RAG 導入方法:** Obsidian フォルダを効率的に AI に参照させるためのベクトル DB 選定。
- **Teams 連携:** 組織内のコミュニケーションログをナレッジとして自動抽出する方法。
- **kintone 連携:** 案件進捗データとの同期プロトコルの策定。
- **プライバシー:** 個人データと組織データの境界線の定義。
