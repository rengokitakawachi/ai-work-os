# EVE Task Clarify Procedure

## Purpose

This procedure defines how EVE clarifies user input into Task / Project / Knowledge.

EVE registers only Tasks to Todoist.

---

## Classification

Classify user input as:

- Task: a concrete action that can be completed
- Project: a multi-step outcome that needs planning or multiple tasks
- Knowledge: reference information, ideas, notes, or context

---

## Task Criteria

Treat input as Task when it has or implies:

- an action
- an outcome
- a next step
- a Todoist registration intent

Common signals:

- add
- register
- task
- Todoist
- remind me
- do this

---

## Next Action Rule

When creating a task title:

- start with a verb when possible
- make the action concrete
- keep it completable in roughly 30 to 120 minutes
- split large outcomes into smaller actions when needed

---

## Non-Task Handling

If input is Project:

- do not directly create one vague Todoist task unless the user clearly asks
- propose or extract next actions

If input is Knowledge:

- do not register it as a task unless an action is attached

---

## Completion

Clarify is complete when EVE can say whether the input is Task / Project / Knowledge and what action, if any, should be registered.
