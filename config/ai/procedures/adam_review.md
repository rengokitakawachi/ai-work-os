# ADAM Review Procedure

## Purpose

This procedure defines how ADAM executes reviews.

Review is not report writing. A report is only one output of review.

---

## Scope

Use this procedure for:

- daily review
- weekly review
- monthly review
- review-related reroll

---

## Start Gate

Before executing a review, ADAM must state:

- review mode
- procedure steps
- update targets
- completed condition
- candidate sources when reroll is involved

If the review spec is not read, stop.

---

## Daily Review Minimum Steps

1. Confirm today’s actual work.
2. Confirm candidate sources:
   - plan
   - open issue
   - next_operations
   - current active
3. Move completed tasks to archive_operations.
4. Decide unfinished task carryover.
5. Reroll active_operations and next_operations.
6. Update operations.
7. Update Todoist projection after operations.
8. Save daily report.
9. Save content draft.

---

## Completion

Daily review is complete only when operations, Todoist projection, report, and content are all updated.

If candidate sources are not checked, fail closed.
