# flow check メモ（軽量）

## 位置づけ

routing / planning / review の骨格が見えてきたので、その次に「どこで流れが止まっているか」をざっと見る機能が必要そう。  
名前は `flow check` がシンプルでよい。

---

## 何を見たいか

- `00_inbox` に未整理入力が溜まっていないか
- `dev_memo` に残ったまま issue / plan / operations に進んでいないものがないか
- `01_issues` に溜まったまま routing されていない論点がないか
- `03_plan` にあるのに `04_operations` に落ちていない重点テーマがないか
- `04_operations` が未完了過多で短期実行順として機能不全になっていないか
- `80_future` に送ったまま戻されていないものがないか

---

## これは何か

review というより、運用の目詰まり検知に近い。

- review = 見直し
- routing = 仕分け
- flow check = 滞留 / 停滞 / 目詰まりの確認

という分け方が自然そう。

---

## 出したいもの

- 今どこが詰まっているか
- 何が滞留しているか
- 何を次に流すべきか
- 何を先に整理しないと全体が悪化するか

---

## 当面の扱い

- 今は issue と dev memo に残す
- いきなり docs や design にしない
- 後で ADAM / EVE 共通の health check 的な層として見直す余地がある
