## Google Antigravity とは 

Google Antigravity は、Googleが開発した「エージェント・ファースト」を掲げる次世代のAIネイティブ開発プラットフォームです。  
これまでのAIエディタが「人間のコーディングを補助する（ペアプログラミング）」存在だったのに対し、Antigravityは**「AIエージェントが自律的にタスクを完結させる（チーム開発）」**ことを前提に設計されています。

### 2つのコア・インターフェース 

Antigravityには、ユーザーの関わり方に応じて切り替えられる2つの画面があります。

**Editor Mode**: 馴染みのあるIDE（統合開発環境）の形式です。AIによるコード補完やチャットを使いながら、人間が主導してコードを書いていく際に使用します。

**Agent Manager Mode**: 複数のAIエージェントを司令塔として管理するダッシュボードで、エージェントが「計画 → 実装 → 検証」という一連のフローを非同期で進める様子をモニタリングできます。

### 「自律的エージェント」による全自動ワークフロー 

Antigravityのエージェントは、エディタ内だけに留まりません。

**プランニング**: タスクを与えると、まず実装手順の「計画書」を作成し、ユーザーに承認を求めます。

**ブラウザ操作**: AIが自ら統合ブラウザを起動。描画テストやUIの挙動確認を行い、エラーがあれば自分で修正を繰り返します。

**ターミナル実行**: 必要なライブラリのインストールやサーバーの起動も、AIが自律的にコマンドを実行します。

### 「Vibe Coding」の実現 

「こういう雰囲気のサイトを作って」「この機能をいい感じに追加して」といった曖昧な自然言語（Vibe）による指示だけで、複雑なロジックを形にできるのが最大の特徴です。 開発者は細かな構文に悩まされることなく、「何を作るか」という本質的な意思決定に集中できるようになります。

## Google Antigravity の導入方法 

Antigravityは現在、Public Previewとして公開されており、誰でも無料で試すことができます。  
初めての方はGoogle公式のチュートリアル[ Getting Started with Google Antigravity ](https://codelabs.developers.google.com/getting-started-google-antigravity?hl=ja)をガイドにするのが最も確実です。  
このCodelabの流れに沿って、私が実際にセットアップした際のポイントをまとめます。

### インストール 

まず、[公式サイト](https://antigravity.google/?hl=ja)から自分の OS（Windows / Mac / Linux）に合ったインストーラーをダウンロードします。  
次に、アプリケーションインストーラを起動し、インストールします。インストールが完了したら、Antigravity アプリケーションを起動します。次のような画面が表示されます。

![](https://cdn-ak.f.st-hatena.com/images/fotolife/p/playjp/20260216/20260216113025.png)

Next をクリックすると次の設定ができます。

**セットアップフローの選択**: 既存の VS Code または Cursor の設定からインポートするオプションが表示されます。インストール後のセットアップウィザードでは、既存の VS Code や Cursor の設定をインポートする選択肢が出ますが、最初は「Fresh Start（新規）」でAntigravity 独自のUIを体験してみるのがおすすめです。

**エディタのテーマ選択**: ライト・ダークなどのテーマを選択できます。

**Antigravityエージェントの権限設定**: エージェントの権限設定を選択できます。詳しくは下記で説明をします。

### エージェントの挙動を決める権限設定 

Antigravity をインストールした後、最も重要なのがこの 「How do you want to use the Antigravity Agent?（エージェントをどのように使用しますか？）」 という設定画面です。

ここでは、AI エージェントに「どこまで勝手に作業をさせて良いか」という権限の範囲を選択します。

![](https://cdn-ak.f.st-hatena.com/images/fotolife/p/playjp/20260216/20260216113003.png)

**プリセットからモードを選択する（左側のパネル）**  
左側のメニューから、自分の開発スタイルに合ったモードを選びます。

**Secure Mode**: 最も制限が強いモードです。AI は提案を行いますが、実際の実行は人間が行います。

**Review-driven development**: AI がコードの修正やコマンド実行を行おうとするたびに、ユーザーへレビュー（承認）を求めます。「AI と対話しながら進める」という現在の主流スタイルに最適です。

**Agent-driven development**: AI の自律性を最大化します。レビューの回数が減り、AI が次々とタスクを完了させていきますが、意図しない変更が行われないよう注意が必要です。

**Custom configuration**: 各項目の権限を個別にカスタマイズします。

**詳細なポリシー設定（右側のパネル）**  
さらに細かく、アクションごとの権限を指定できます。

**Terminal execution policy**: ターミナルでのコマンド実行（ライブラリのインストールやサーバー起動など）の権限。

**Review policy**: 変更内容をどの程度ユーザーに確認させるかの頻度。

**JavaScript execution policy**: JavaScript コードの実行権限。

### 事前設定（日本語化・拡張機能） 

快適に使うために、以下の2点を設定しておきましょう。

**日本語化**: 拡張機能で「Japanese Language Pack」をインストールします。また、AIチャットの応答を日本語に固定するため、Settings ＞ Customizations に「応答はすべて日本語で行ってください」と書き足すのがコツです。

**Antigravity Bridge（Chrome拡張）**: これを導入することで、AIがブラウザを立ち上げ、UIの崩れやコンソールエラーを「目で見て」判断し、自己修復するループが可能になります。

### 実際に実装をしてみる 

今回は、以前から私が「こんなのあったら便利だな」と考えていた、高機能なタスク管理ツールを実際に作成していこうと思います。  
単なるToDoリストではなく、業務でも使用できるカンバン方式かつ工数管理ができるツールを、Antigravityのエージェントにお願いして作れるか挑戦です。 今回使用したモデルは、Claude Sonnet 4.5です。

### 実装したいものの構成を伝える 

まずはチャット欄で、頭の中にある構想をAIに伝えます。  
今回は以下のようにAIに伝えました。

> 自分専用のタスク管理ツールを作成したい。  
・Trelloのようなドラッグ＆ドロップができるカンバンボード  
・それぞれのタスクの中でさらにチェックマーク式で子タスクをかける  
・その子タスクに作業時間を測れるストップウォッチをつけて、開始日・終了日作業時間を記録したい

### 自動実装 

指示を出すと、AIが即座に実装計画を提示します。内容を確認して承認すると、そのまま実装がスタート。

![](https://cdn-ak.f.st-hatena.com/images/fotolife/p/playjp/20260216/20260216113007.png)

### 自動デバッグ 

コードが書けた後、AIが自動的に動作確認事項を考え、ブラウザを立ち上げてテストを開始します。

![](https://cdn-ak.f.st-hatena.com/images/fotolife/p/playjp/20260216/20260216113012.png)

### 実装完了 

作りたいものを伝えてから、ものの5分で実装が完了しました。

デザインの微調整は必要ですが、ドラッグ&ドロップやタイマーといった複雑な機能も、指示通り正確に動いています。 またAntigravityでは、実際に使っていく中で「もっとこうしたい」と思った機能をリクエストするだけで、**驚くほど簡単にアップデート**を行うことができます。  
私もこのツールに後からカラーモード（ダーク/ライト）の切り替えや、カレンダー機能を実装しました。これにより、その日どのタスクに何時間費やしたかを視覚的に振り返ることができるようになっています。 現在はデータの保存先として Local Storage を使用しているため、自分のブラウザ内ですべてが完結するシンプルな構成です。今後は、Firebase などのバックエンドと連携させて、複数デバイスで同期できるように拡張していく予定です。

## 実際に触って分かった「Antigravity」 

これまでのAIは、チャット欄で「ここを直して」と指示する、いわば単発の作業をこなす存在でした。一方でAntigravityでは、「認証機能」「デザイン」「DB設計」といった役割を、複数のAIエージェントに並列で任せることができます。また、自分はコードを書くのではなく、AIから上がってくる計画書をレビューして、承認ボタンを押すだけ。まるで、エンジニアというより「テックリード」になったような感覚でした。一番驚いたのは、デバッグのプロセスです。 「エラーが出たから直して」と言うと、AIが勝手にローカルサーバーを立て、自らブラウザを操作して挙動を確認し、コンソールログを見て修正を完結させてしまいます。 人間がエラー内容を貼るという手間が一切消えたのは革命的です。

これまでのAIは、チャット欄で「ここを直して」と指示する、いわば単発の作業をこなす存在でした。一方でAntigravityでは、「認証機能」「デザイン」「DB設計」といった役割を、複数のAIエージェントに並列で任せることができます。また、自分はコードを書くのではなく、AIから上がってくる計画書をレビューして、承認ボタンを押すだけ。まるで、エンジニアというより「テックリード」になったような感覚でした。一番驚いたのは、デバッグのプロセスです。 「エラーが出たから直して」と言うと、AIが勝手にローカルサーバーを立て、自らブラウザを操作して挙動を確認し、コンソールログを見て修正を完結させてしまいます。 人間がエラー内容を貼るという手間が一切消えたのは革命的です。
