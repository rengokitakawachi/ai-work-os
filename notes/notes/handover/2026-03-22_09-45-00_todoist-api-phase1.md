# Handover

## 概要
Phase1のTodoist API新アーキテクチャ実装と実動検証を実施。create / list / validation / subtasks / close まで一連のフローを実機確認し、旧APIラッパー化の方針を確立。

---

## 成功（Success）
- tasks-v2 API（新アーキテクチャ）の実装完了
- handler / validate / dispatch / service / client の分離構造が成立
- create API 実動確認（Todoist登録成功）
- list API 実動確認（50件制限含め正常）
- validation error がdocs仕様通りに返却
- subtasks 作成成功（親子関係含む）
- task-close をラッパー化し新service経由で完了処理成功
- エラー構造が repo-resource 準拠で正常動作

---

## 判明事項（Findings）
- Vercel無料版はAPI数12制限 → 新API追加は不可
- Todoist APIはデフォルトで約50件制限（limitで拡張可能）
- Todoist APIのcreateはレスポンスが空になるケースあり
- upstreamエラーをserviceで潰すとデバッグ困難 → 透過が必要
- close処理は /tasks/{id}/close で正しく動作
- APIは増やすのではなく「既存をラッパー化」が正解

---

## 失敗 / 未解決（Issues）
- /api/task-update が未ラッパー化
- /api/task（create旧API）が未ラッパー化
- tasks-v2 と tasks の二重構造が残っている
- API数が上限に達しているため整理が必要
- assignee の Todoist変換未実装（Phase1では許容）

---

## 次のアクション（Next Actions）
