## 判明事項（Findings）
- `future` は archive と近い位置づけだが、意味は別である
- 実フォルダ名は `80_future/` として扱う方針が整理済み
- completed / superseded / split 元は archive、deferred は future へ送る考え方が整理済み
- review は `intake / daily / weekly / monthly` に分けるのが自然
- design review は monthly review の一部として扱うのが妥当
- report は review の結果を保存する成果物として切り分けると整合しやすい
- `03_plan` は active な plan のみを置く前提で整理済み
- `docs/05_roadmap.md` は新方針へ更新済み
- `notes bulk` は実装済み
- `code bulk` は未実装
- `docs/10_repo_resource_api.md` は `notes bulk` 実装後の内容にまだ未反映
- `adam_instruction.md` では `docs > notes > code` のような単純序列より、
  `docs = 仕様判断の正本 / notes = 補助情報 / code = 実装実態`
  の表現が自然
