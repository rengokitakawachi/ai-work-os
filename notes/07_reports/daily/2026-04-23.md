- 最小 split 単位は `1見出し = 1item` とする
- `まとめ` / `summary` は split item にしない
- split に失敗したときは、既存の `1ファイル = 1item` を維持する
- daily review では candidate source 明示 → reroll → operations 更新 → Todoist projection 更新 → report → content の順を守る
