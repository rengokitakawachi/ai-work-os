- `AI/` は正規配置ではなく旧配置として扱う

## 学び / 気づき

- README のような入口文書も、確認と完了反映まで含めて当日作業として扱う方が実感に合う
- instruction の保存場所を access path に乗せるだけで、確認・更新フローの安定性がかなり上がる
- 形式統一は有効だが、責務まで統一すると役割が崩れる

- EVE のような運用人格は、機能を足すよりも不要機能を持たせない設計が重要である
- 日報は起案日ではなく、当日実行したイベントで記録する方がズレが起きにくい
- instruction の自動同期は保守性とパフォーマンスの両立が難しく、固定 Instructions と repo 正本の役割分離が重要である
