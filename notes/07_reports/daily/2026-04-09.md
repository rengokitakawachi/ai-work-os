## 重要な意思決定

- 今回の Todoist 連携は双方向同期ではなく、ADAM が operations 正本を更新し、その結果を Todoist execution view に片方向投影するプロトタイプとして扱う
- projection 対象は当面 `active_operations` のみとし、`next_operations` は次段に回す
- `todoist_task_id` は別 state ではなく operations task 本文の `external` に持つ
- close は独立 action として扱うが、client 実装上は `updateTask(..., { completed: true })` 互換でよい
- `project_id` は ADAM 専用品ではなく、projection target project を指定する汎用入力とする
- ADAM は projection 時に既存 `createTask` / `updateTask` を task ごとに直叩きせず、専用 projection Action を優先する
- projection の検証はまず `dry_run` を前提に進める

## 学び / 気づき

- 既存 tasks API / Todoist client は完成形ではないが、projection service を差し込む「途中形」としては十分使える
- projection の責務を service に集約し、API を薄く保つ方針は既存 design と整合する
- 最初に設計だけで理想形を切るより、現状 repo に接続可能な最小差分へ寄せた方が前進が速い
- `project_id` を task 本文に持たせず実行コンテキストで渡す形にしたことで、投影先分離と汎用性の両立がしやすくなった
- iPhone では schema 編集ができないため、Action 追加や更新の最終反映は web 版前提で考える必要がある
