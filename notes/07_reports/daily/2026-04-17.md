## 重要な意思決定

- `design review` は棚卸し・候補抽出に限定し、docs / design / future / archive / operations candidate の行き先判定は `design routing` に分離する
- `design routing` では `archive → future → docs → operations → design` の優先順 rule を採用する
- `docs_ready` は `operations` より優先し、operations は docs 昇格の代替にしない
- `routeDesignCandidates(...)` の最小出力は `mode / routed_design_candidates / action_plan` とする
- `route_to: design` は retained no-op とし、routing のたびに design file 本体を mutate しない
- `route_to: docs` でも docs へ直接 apply せず、docs candidate 化までに留める
- `route_to: operations` は active_operations を直接更新せず、queue payload に留める
- 2026-04-17 の daily review では完了 task の archive 移動は行わず、Day0 task を継続扱いで先頭維持する
