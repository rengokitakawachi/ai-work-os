- Strategy Domain 中心設計の確立

- docs / notes / code の役割分離を明確化

---

## 主要な意思決定

- Domain中心設計（外部ツール非依存）
- docs を SSOT とする原則の徹底
- operations を短期実行順の正本とする
- inbox は未整理入力専用とし、処理後は archive へ移動
- future は「将来OS」として独立レイヤー化
- archive は active と同一構造のスナップショットとする
- source_ref は出自追跡専用の最小メタデータとする
- インテークレビューを構造変換プロセスとして定義
