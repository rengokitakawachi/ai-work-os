# notes/README.md

## 目的

notes は
iPhone やブラウザから蓄積した未確定情報を保管し
機能検討と仕様検討につなげるための作業領域である。

---

## 位置づけ

notes は思考レイヤーである

SSOT ではない

正式仕様は docs に置く

---

## 基本フロー

外部入力
↓
inbox に蓄積
↓
exploration で機能検討
↓
design で仕様検討
↓
docs に昇格

---

## ディレクトリ

inbox
exploration
design
decisions
backlog
logs

---

## 運用原則

・まず inbox に入れる
・未確定のまま docs に書かない
・機能案は exploration で整理する
・仕様案は design に集約する
・判断結果は decisions に残す
・次にやることは backlog で管理する
